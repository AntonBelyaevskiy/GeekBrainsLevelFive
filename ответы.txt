1)	Как вы собираетесь передавать файлы? [поток байтов/свой протокол передачи, сериализация, либо ваш вариант]
Передавать файл планирую и при помощи сериализации и потока байтов.

Название и расширение файла передаю при помощи сериализации
объект, у которого будет:
имяФайла (String поле)
расширениеФайла (String поле)
На клиенте файл превратили в объект и сериализавали, передали его на сервер, на сервере файл десериализавали
и создали новый файл на основе этого объекта.

После того как убедились, что файл с нужным именем и расширением создан.

Передаем содержимое из нашего файла на клиенте, в наш файл на сервере при помощи потока байтов.
Эти операции будут производиться как транзакция:
start
создали файл
заполнили файл содержимым
end



2)	Нужна ли вам база данных?
Нужна для хранения пользователей(учетных данных), в принципе думаю можно хранить эти данные на сервере не использую БД,
но с БД будет более удобно и безопасно.



3)	Как вы собираетесь давать команды серверу? (на переименование файла, удаление файла, список файлов и т.д.)
Планирую использовать "/" перед названием команды
/rename имяФайла новоеИмяФайла
/remove
/list
...
Вход в систему будет так же происходить по команде /login логин пароль

Таким образом на сервер будeт прилетать 3 разных сущности, которые нужно будет различать между собой.
1.Файл.расширение
2.Содержимое
3.Команды
